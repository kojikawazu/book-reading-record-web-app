## 第1部: UI / UX 仕様

### 1. 対象
- アプリ名: 読書記録Webアプリ（仮）
  - 実装シェル（`OrganicShell`）のブランド表記は「LeafLog」。ダッシュボード本文の見出しは「読書記録アプリ」。
- 対象ユーザー: 単一ユーザー（本人利用）
- 利用デバイス: モバイル優先、デスクトップ対応

### 2. 画面一覧
1. ダッシュボード (`/`)
2. 書籍登録 (`/books/new`)
3. 書籍詳細・進捗記録 (`/books/[id]`)
4. 統計レポート (`/stats`)
5. 週次サマリー（ダッシュボード内セクション）
6. ログイン (`/auth/login`)（`supabase` モードの Google OAuth ログイン）

### 3. レイアウト詳細

#### 3.0 共通レイアウト（アプリシェル `OrganicShell`）
全画面は共通シェル `OrganicShell` でラップされる。
- 左サイドバー（デスクトップ表示。モバイルでは本文上部に横並びのナビ）
  - ナビゲーション: ホーム (`/`) / 書籍登録 (`/books/new`) / 統計レポート (`/stats`)
  - 現在ドライバー表示（`Supabase Auth / Google OAuth` または `localStorage / Playwright E2E`）
- ヘッダー
  - 画面タイトル・サブタイトル
  - ヘッダー内検索ボックスは**装飾用（`readOnly`・非機能）**。実際の検索はダッシュボード本文の検索パネルで行う
  - `supabase` モード時はログイン／ログアウトボタンを表示
- フッター（デスクトップ表示のみ、ブランド表記）

#### 3.1 ダッシュボード
- ヒーローセクション
  - 「新しい記録を始める」CTA（`/books/new` へ遷移する書籍追加導線）
- 検索パネル（機能版）
  - 書籍タイトル・著者・タグの部分一致検索
- セクション
  - 読書前一覧
  - 読書中一覧
  - 保留一覧
  - 完読一覧
  - 感想未記入（完読済みで感想未登録、または感想3項目が空）
  - 週次サマリー（直近7日の読了ページ数、進捗記録回数、感想数）
- サイドバー（本文内）
  - 今週のバッジ: 継続記録（今週3回以上の進捗記録）/ ページハンター（今週100ページ以上）/ 感想投稿（今週1件以上）/ 完読達成（累計1冊以上）の達成状況を表示
  - 運用メモ（補足説明）
- 補足表示
  - 総ページ数が不正（`totalPages <= 0`）な書籍は一覧から除外し、件数を警告表示する（本書 第2部 §6 表示計算）
  - `local` モードで破損データを検知・初期化した場合は復旧メッセージを表示する

#### 3.2 書籍登録
- 入力項目
  - タイトル（必須）
  - 著者（必須）
  - ジャンル（任意）
  - 読書形式（必須: 紙/電子/音声）
  - 総ページ数（必須、正の整数）
  - タグ（任意、カンマ区切り）
  - 初期ステータス（必須: 読書前/読書中/保留、デフォルトは読書前）
- 保存時の内部設定
  - `currentPage` は `0` で初期化する
- アクション
  - 保存
  - キャンセル

#### 3.3 書籍詳細・進捗記録
- 書籍情報カード
  - タイトル、著者、形式、総ページ数、現在ページ、進捗率、ステータス
- 進捗記録フォーム
  - 読了ページ（今回到達ページ、必須）
  - メモ（任意）
  - ステータス（読書前/読書中/保留/完読）
  - 記録日（デフォルト当日）
- 進捗履歴
  - 日付、ページ、メモ、ステータス
- 完読導線
  - 現在ページが総ページ以上で完読候補
  - 完読保存時に感想入力セクションを表示

#### 3.4 完読時感想
- 表示タイミング
  - 完読保存時のみ
- 入力項目
  - 学び
  - 次に行動すること
  - 印象に残った一文
- バリデーション
  - 文字数下限なし（空入力可）
- 保存後
  - 書籍ステータスを「完読」に変更
  - ダッシュボードの完読一覧へ反映

#### 3.5 再読
- 完読済み書籍を再読状態（読書中）へ戻せる
- 再読開始時は現在ページを `0` にリセットする
- 再読開始後はページを直接入力で更新可能

#### 3.6 統計レポート
- 表示内容
  - KPIカード: 登録書籍数 / 完読率 / 今週の進捗記録 / 今週の読了ページ
  - ステータス分布（読書前/読書中/保留/完読、件数と割合バー）
  - 読書形式の内訳（紙/電子/音声、件数と割合バー）
  - 補助カード: 直近7日の感想数 / 読書中の冊数 / 保留中の冊数
- データソース
  - `NEXT_PUBLIC_REPOSITORY_DRIVER=supabase`: `/api/book-record/*` 経由でSupabase（Prisma）から取得
  - `NEXT_PUBLIC_REPOSITORY_DRIVER=local`: localStorage の `book-reading-record.v1` から取得（E2E受け入れで使用）

### 4. UX方針
- 進捗入力は30秒以内完了を目標（性能観点は `docs/04-non-functional-specification.md` 参照）
- モバイルで片手入力できるフォーム密度
- エラーメッセージは項目直下に表示

### 5. 関連仕様参照
- データモデルと保存仕様は `docs/05-data-specification.md` を参照する
- 入力検証とセキュリティ要件は `docs/06-security-specification.md` を参照する

## 第2部: ビジネスロジック（業務ルール）

### 1. 目的
- 状態遷移、表示判定、集計、並び順の業務ルールを定義する

### 2. ステータス遷移
- 利用ステータス
  - 読書前（`not_started`）
  - 読書中（`reading`）
  - 保留（`paused`）
  - 完読（`completed`）

- 基本遷移
  - `not_started -> reading`
  - `reading -> paused`
  - `paused -> reading`
  - `reading -> completed`
  - `paused -> completed`

- 完読判定
  - `currentPage >= totalPages` のとき、保存時に `status=completed` を強制する
  - `currentPage < totalPages` かつ `status=completed` は保存エラー

### 3. 再読ルール
- 完読済み書籍は `completed -> reading` を許可する
- 再読開始時:
  - `currentPage = 0`
  - `completedAt = undefined`
  - `reflection` は保持（初期化しない）
- 再完読時:
  - `reflection` は再編集可能（上書き保存を許可）

### 4. 感想未記入判定
- 対象は `status=completed` の書籍
- 次のいずれかで未記入扱い
  - `reflection` が未登録
  - `learning` / `action` / `quote` がすべて空文字（trim後）

### 5. 週次集計ルール
- 集計対象期間: 直近7日（当日含む）
- 日付判定: クライアントのローカルタイムゾーン
- 指標
  - 読了ページ数:
    - 期間内最初のログは `0` を起点に差分計算する
    - 2件目以降は同一書籍の連続ログ間差分で計算する
    - 負数差分は `0` として扱う
  - 進捗記録回数: 期間内 `ProgressLog` 件数
  - 感想数: 期間内 `Reflection` 作成件数

### 6. 並び順
- ダッシュボード各書籍一覧
  - `updatedAt` 降順
  - 同値時は `createdAt` 降順
  - さらに同値時は `id` 昇順
- 進捗履歴
  - `loggedAt` 降順
  - 同値時は `id` 昇順
- 検索結果
  - 絞り込み後に書籍一覧と同じ並び順を適用

### 7. 表示計算
- 進捗率: `currentPage / totalPages * 100` を小数第1位四捨五入
- `totalPages <= 0` のデータは表示対象外としてエラー表示
