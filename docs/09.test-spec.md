# Test Specification

## 1. 目的
- Playwright E2Eの実行条件と運用ルールを定義する

## 2. テスト対象
- E2Eのみ（単体テストは作成しない）
- 詳細ケース（Case 1-17）は `docs/04.e2e-cases.md` を参照

## 3. 実行環境
- ブラウザ: Chromium
- 実行ディレクトリ: `front/`
- コマンド:
  - `pnpm test:e2e`
- 合格条件:
  - 全ケース成功（skipなし）

## 4. 実行前処理
- 各テスト開始時に `localStorage` を初期化する
- 必要な初期データはテスト側で投入する

## 5. セレクタ方針
- `data-testid` を優先して要素を取得する
- 文言ベースの取得は補助的に利用する

## 6. テスト観点
- 正常系: 登録、進捗、完読、再読、検索、週次可視化
- 異常系: バリデーション、破損データ復旧
- 永続化: localStorageキー・リロード耐性
- 並び順: 書籍一覧・進捗履歴
