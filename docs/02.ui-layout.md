# UI Layout

## 1. 対象
- アプリ名: 読書記録Webアプリ（仮）
- 対象ユーザー: 単一ユーザー（本人利用）
- 利用デバイス: モバイル優先、デスクトップ対応

## 2. 画面一覧
1. ダッシュボード (`/`)
2. 書籍登録 (`/books/new`)
3. 書籍詳細・進捗記録 (`/books/[id]`)
4. 統計レポート (`/stats`)
5. 週次サマリー（ダッシュボード内セクション）

## 3. レイアウト詳細

### 3.1 ダッシュボード
- ヘッダー
  - アプリ名
  - 「書籍を追加」ボタン
- 検索バー
  - 書籍タイトル・著者・タグの部分一致検索
- セクション
  - 読書前一覧
  - 読書中一覧
  - 保留一覧
  - 完読一覧
  - 感想未記入（完読済みで感想未登録、または感想3項目が空）
  - 週次サマリー（進捗記録回数、読了ページ数、感想数）

### 3.2 書籍登録
- 入力項目
  - タイトル（必須）
  - 著者（必須）
  - ジャンル（任意）
  - 読書形式（必須: 紙/電子/音声）
  - 総ページ数（必須、正の整数）
  - タグ（任意、カンマ区切り）
  - 初期ステータス（必須: 読書前/読書中/保留、デフォルトは読書前）
- 保存時の内部設定
  - `currentPage` は `0` で初期化する
- アクション
  - 保存
  - キャンセル

### 3.3 書籍詳細・進捗記録
- 書籍情報カード
  - タイトル、著者、形式、総ページ数、現在ページ、進捗率、ステータス
- 進捗記録フォーム
  - 読了ページ（今回到達ページ、必須）
  - メモ（任意）
  - ステータス（読書前/読書中/保留/完読）
  - 記録日（デフォルト当日）
- 進捗履歴
  - 日付、ページ、メモ、ステータス
- 完読導線
  - 現在ページが総ページ以上で完読候補
  - 完読保存時に感想入力セクションを表示

### 3.4 完読時感想
- 表示タイミング
  - 完読保存時のみ
- 入力項目
  - 学び
  - 次に行動すること
  - 印象に残った一文
- バリデーション
  - 文字数下限なし（空入力可）
- 保存後
  - 書籍ステータスを「完読」に変更
  - ダッシュボードの完読一覧へ反映

### 3.5 再読
- 完読済み書籍を再読状態（読書中）へ戻せる
- 再読開始時は現在ページを `0` にリセットする
- 再読開始後はページを直接入力で更新可能

### 3.6 統計レポート
- 表示内容
  - 登録書籍数、完読率
  - 直近7日の読了ページ数、進捗記録回数、感想数
  - ステータス分布（読書前/読書中/保留/完読）
  - 読書形式分布（紙/電子/音声）
- データソース
  - Phase 1 は localStorage の `books` / `progressLogs`

## 4. UX方針
- 進捗入力は30秒以内完了を目標
- モバイルで片手入力できるフォーム密度
- エラーメッセージは項目直下に表示
