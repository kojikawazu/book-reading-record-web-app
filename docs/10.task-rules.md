# Implementation Rules And Roles

## 1. 目的
- MVP実装の運用ルールと役割分担を定義する
- 「タスク管理」は `docs/11.implementation-tasks.md` に分離し、本ドキュメントでは扱わない

## 2. 役割定義
### 2.1 設計者
- 仕様変更の判断責任を持つ
- `docs/02`〜`docs/11` の整合性を担保する
- 実装開始前に「仕様確定」を宣言する

### 2.2 実装者
- `front/` のみを変更する
- 実装は `docs/03` と各詳細仕様（`docs/05`〜`docs/11`）に従う
- E2Eが通る状態でPRを作成する

### 2.3 レビュー者
- 仕様適合、挙動、リスク、テスト網羅を確認する
- 指摘は再現手順付きで記載する
- 受け入れ基準は `docs/04.e2e-cases.md` を正とする

## 3. 進行フロー
1. 設計者が対象タスクを選定し、仕様確定を宣言
2. 実装者が機能実装とE2E追加を実施
3. レビュー者が仕様適合レビューとE2E結果を確認
4. 指摘修正後、再レビューして完了

## 4. レビューチェックリスト
- `base/` に変更がない
- `front/` のみ変更されている
- `docs/04` のCase 1-18が再現またはE2Eで通過
- データ破損時にクラッシュしない
- 再読時の `currentPage=0` / `completedAt` 解除 / `reflection` 保持が満たされる
- 再完読時に `reflection` が再編集（上書き保存）できる
- 並び順と週次集計が `docs/06` と一致する
- ユーザー/プロフィールのデータモデルがMVPに混入していない

## 5. 進捗管理ルール
- タスク状態は `todo` / `doing` / `review` / `done` の4状態
- 1PRで扱うタスクは最大2件まで
- 仕様変更が必要になった場合は、実装前に設計者へ差分提案する
- タスク追加/完了時は `docs/11.implementation-tasks.md` を更新する
