# 要求仕様書（Business Requirements）

プロジェクトの背景・目標・スコープ・制約を定義する。詳細な機能要件は `docs/02-requirements-specification.md` を参照する。

## 1. 背景・目的

- 読書記録（ページ進捗・完読感想・再読）を管理する単一ユーザー向け MVP。
- 進捗・完読・感想・再読までを一気通貫で管理し、読書記録を習慣化することを目的とする。

## 2. 対象ユーザー

- 単一ユーザー（本人利用）前提。
- ユーザープロフィール機能は MVP 対象外（必要時に追加検討）。

## 3. スコープ（MVP 提供機能）

- 書籍登録（タイトル/著者/形式/総ページ/初期ステータス）
- ページ進捗入力（絶対値）+ メモ + ステータス更新
- 完読時感想（学び/行動/一文、空入力可）
- 再読（完読 → 読書中、`currentPage=0`）
- ダッシュボード（読書前/読書中/保留/完読/感想未記入）
- 検索（タイトル・著者・タグ）
- 週次サマリー（読了ページ/進捗回数/感想数）
- 統計レポート（`/stats`）
- 認証（Supabase Auth / Google OAuth）

## 4. 決定事項（確定）

- 単一ユーザー前提
- 進捗管理指標はページ数
- 完読条件は `現在ページ >= 総ページ`
- 完読時のみ感想入力を表示
- 感想3項目は空入力可（文字数下限なし）
- 進捗記録にはメモ項目を含める
- ステータス: `読書前` / `読書中` / `保留` / `完読`
- 完読後に再読（`読書中`へ戻す）を許可
- 技術スタック: Next.js / TypeScript / Tailwind CSS
- 認証: Supabase Auth（Google OAuth）
- E2Eテスト: Playwright
- データ永続化: Supabase（通常運用）/ localStorage（`local` モード）
- ユーザープロフィール機能はMVP対象外（必要時に追加検討）
- ローカル開発優先（まずローカルで動作）
- デプロイ先: Vercel

## 5. 制約・前提

- 実装は `front/` のみ。
- `base/` は参照用の既存 MVP であり **read-only**（編集禁止）。
- ローカル開発環境で即起動可能であること。

## 6. 禁止事項

- `base/` 配下の編集
- このリポジトリから Prisma `db push` / `migrate` を実行すること
- 単体テスト追加（E2E のみ。`docs/08-test-specification.md` 参照）
